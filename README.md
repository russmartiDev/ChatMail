# ChatMail AI Backend

Node.js + Express backend for ChatMail AI. Handles Gmail OAuth, email fetching, and AI processing.

## Setup

### 1. Install dependencies
```bash
npm install
```

### 2. Configure environment
```bash
cp .env.example .env
```
Then fill in `.env`:
- `JWT_SECRET` — any long random string
- `ANTHROPIC_API_KEY` — from https://console.anthropic.com
- `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET` — from https://console.cloud.google.com
  - Enable Gmail API
  - Create **Web Application** OAuth 2.0 credentials
  - Add `http://localhost:3000/auth/gmail/callback` as Authorized Redirect URI

#### Database configuration
This backend requires a PostgreSQL database; SQLite support has been dropped.
Set the `DATABASE_URL` environment variable with a full connection string.
The application will refuse to start if the value is missing.

### 3. Run
```bash
# Development (auto-restart)
npm run dev

# Production
npm start
```

## Database & Embeddings

A new `email_vectors` table holds embeddings for RAG search.  It's defined
using Postgres's `vector` extension and looks like:

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE email_vectors (
    id TEXT PRIMARY KEY,
    content TEXT NOT NULL,
    metadata JSONB,
    embedding VECTOR(3072) NOT NULL
);
```

The `content` field is generated by Gemini from each email (subject, sender,
body) to produce a searchable summary; this makes the vectors more useful in a
retrieval‑augmented search scenario.

To recreate and populate the table, run:

```bash
cd mailmind-backend
node scripts/embed_emails.js
```

The script automatically installs the `vector` extension, drops the old
`email_vectors` table, and creates it with correct column types; it then
loops over all emails, generating a RAG‑friendly summary via Gemini and
storing the resulting vector (formatted as `[v1,v2,…]`).  **Every execution
starts from scratch**, so run it only when you're ready to rebuild the index
– you'll need to adapt it for incremental updates if you prefer not to wipe
data.

## API Endpoints

### Auth
| Method | Path | Description |
|--------|------|-------------|
| POST | `/auth/register` | Create account `{ email, password, name }` |
| POST | `/auth/login` | Login `{ email, password }` → returns JWT |
| GET | `/auth/me` | Get current user (requires JWT) |

### Gmail
| Method | Path | Description |
|--------|------|-------------|
| GET | `/auth/gmail/url` | Get Google OAuth URL to open in browser |
| GET | `/auth/gmail/callback` | OAuth callback (Google redirects here) |
| GET | `/auth/gmail/status` | Check if Gmail is connected |
| DELETE | `/auth/gmail` | Disconnect Gmail |

### Emails
| Method | Path | Description |
|--------|------|-------------|
| POST | `/emails/sync` | Fetch Gmail + AI process → cache result |
| GET | `/emails` | Return cached processed emails |
| DELETE | `/emails` | Clear cached emails |

## Architecture

```
Mobile App
   │
   ├── POST /auth/login → JWT token
   │
   ├── GET /auth/gmail/url → opens browser
   │        └── Google OAuth → /auth/gmail/callback
   │                └── tokens saved to user
   │
   └── POST /emails/sync
            ├── fetchEmails() → Gmail API (using stored tokens)
            └── processEmails() → Claude AI
                     └── returns categorized batch
```

## Testing with curl

```bash
# Register
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"password123","name":"Test User"}'

# Login
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"password123"}'

# Get Gmail OAuth URL (replace TOKEN)
curl http://localhost:3000/auth/gmail/url \
  -H "Authorization: Bearer TOKEN"

# Sync emails
curl -X POST http://localhost:3000/emails/sync \
  -H "Authorization: Bearer TOKEN"
```
